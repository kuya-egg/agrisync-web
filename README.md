# Agrisync Landing Page + Product README

## Product Name

Agrisync

## Tagline

Your Smart Farm Assistant

## Core Mission

Agrisync is an AI-powered smart farming platform designed for Filipino farmers. It guides farmers through the entire farming cycle — from crop planning, planting, monitoring, disaster preparation, harvesting, and selling.

The platform combines:

* AI-powered guidance through Anie (virtual farm assistant)
* Localized crop recommendations
* Crop monitoring with estimated harvest timelines
* Personalized action items and climate alerts
* DA-based market price insights
* Community support and expert-backed discussions
* Downloadable farming activity reports

The goal is to replace guesswork and scattered advice with simple, localized, practical farming assistance.

# Development Architecture Notes

## Data Handling Rules

The entire landing page must avoid hardcoded repeated UI values.

All repeatable content should be driven through centralized TypeScript objects and arrays.

Examples include:

* Navigation links
* Pricing plans
* FAQ items
* Feature cards
* Testimonials
* Statistics
* CTA sections
* Screenshots
* Farming phases
* Social links
* Footer links

## Required Structure Style

Use structured TypeScript configuration objects such as:

```ts
export const pricingPlans = [
  {
    name: 'Farmer Plus',
    price: '₱149/month',
    features: [],
    recommended: true,
  }
]
```

```ts
export const features = [
  {
    title: 'Crop Monitoring',
    description: '...',
    image: '...',
  }
]
```

## Benefits

This architecture allows:

* Easier maintenance
* Cleaner components
* Faster future scaling
* CMS migration readiness
* Localization support
* Easier pricing updates
* Dynamic rendering

## Component Rules

Components should:

* Receive props
* Render dynamically from arrays
* Avoid duplicated markup
* Separate content from presentation
* Be reusable and composable

Avoid:

* Massive monolithic components
* Repeated hardcoded cards
* Inline repeated copy
* Static duplicated layouts
